# 纸巾筒小游戏系统设计文档

## 1. 游戏概述

### 1.1 游戏名称
纸巾筒小游戏（Tissue Game）

### 1.2 游戏类型
模拟休闲小游戏

### 1.3 核心玩法
模拟从纸巾筒抽取纸巾的过程，包含抽取、切断、清理、装填等机制，考验玩家的操作技巧和节奏感。

### 1.4 目标平台
- Unity 编辑器（PC）
- Android 移动设备

---

## 2. 游戏机制

### 2.1 基本流程

```
┌─────────────┐
│   游戏开始   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  向下划动   │ ──────► 抽取纸巾
│  (抽取)     │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  横向划动   │ ──────► 切断纸巾
│  (切断)     │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  纸巾堆积   │
│  (下方区域) │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  横向划动   │ ──────► 清理纸巾 (3 次/个)
│  (清理)     │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  纸巾耗尽？ │ ──是──► 清空弹仓 ──► 装填新卷
└─────────────┘
```

### 2.2 操作方式

| 操作 | 输入方式 | 判定区域 | 效果 |
|------|----------|----------|------|
| 抽取 | 向下划动 | 纸巾筒区域 | 纸巾延伸 |
| 切断 | 向左/右划动 | 纸巾筒区域 | 纸巾切断并堆积 |
| 清理 | 向左/右划动 | 堆积区域 | 累计 3 次清除一个纸巾 |
| 清空 | 向右划动 | 纸巾筒区域 | 清空弹仓（耗尽后） |
| 装填 | 向左划动 | 纸巾筒区域 | 装填新纸卷（清空后） |

### 2.3 输入处理

**统一输入系统：**
- PC：鼠标左键拖动
- Android：手指触摸滑动

**输入检测流程：**
1. 检测按下位置所在的判定区域
2. 跟踪拖动轨迹
3. 根据拖动距离和方向判断操作类型
4. 触发相应事件

## 3. 系统架构

### 3.1 组件结构

```
TissueGameManager (主管理器)
├── TissueBox (纸巾筒)
├── TissuePaper (纸巾)
├── TissuePileManager (堆积管理)
└── TissueInputHandler (输入处理)
```

### 3.2 组件职责

#### 3.2.1 TissueGameManager
- 游戏整体状态管理
- 游戏流程控制
- 得分计算
- 组件协调

#### 3.2.2 TissueBox
- 纸巾筒状态管理
- 抽取响应
- 切断响应
- 装填流程

#### 3.2.3 TissuePaper
- 纸巾延伸控制
- 切断效果
- 收回动画

#### 3.2.4 TissuePileManager
- 堆积纸巾管理
- 清理逻辑
- 数量限制检查

#### 3.2.5 TissueInputHandler
- 输入检测
- 判定区域管理
- 划动方向识别

### 3.3 数据流

```
用户输入
    │
    ▼
TissueInputHandler
    │
    ├──► TissueBox (抽取/切断/装填)
    ├──► TissuePaper (延伸/切断)
    └──► TissuePileManager (清理)
    │
    ▼
TissueGameManager (状态同步、得分计算)
```

## 4. 状态机设计

### 4.1 游戏状态机 (TissueGameState)

```
                    ┌──────────┐
                    │   Idle   │
                    └────┬─────┘
                         │ StartGame()
                         ▼
                    ┌──────────┐
             ┌─────►│ Playing  │◄─────┐
             │      └────┬─────┘      │
             │           │            │
      ┌──────┴──────┐    │      ┌─────┴─────┐
      ▼             ▼    │      ▼           ▼
┌───────────┐ ┌─────────┐ │ ┌──────────┐ ┌──────────┐
│  Pulling  │ │ Cutting │ │ │  Empty   │ │ GameOver │
└─────┬─────┘ └────┬────┘ │ └────┬─────┘ └──────────┘
      │            │      │      │
      └────────────┴──────┘      │
                                 │
                    ┌────────────┴────────────┐
                    │                         │
                    ▼                         ▼
              ┌───────────┐           ┌───────────┐
              │Clearing   │──────────►│ Reloading │
              │Chamber    │  向左划动  │           │
              └───────────┘           └───────────┘
```

### 4.2 纸巾筒状态机 (TissueBoxState)

| 状态 | 进入条件 | 退出条件 | 可执行操作 |
|------|----------|----------|------------|
| Idle | 初始状态/装填完成 | 向下划动 | 抽取、切断 |
| Pulling | 向下划动 | 0.2 秒后 | - |
| Empty | 纸巾耗尽 | 向右划动 | 清空弹仓 |
| ClearingChamber | 向右划动 | 动画完成 | - |
| Reloading | 向左划动 | 动画完成 | - |

### 4.3 纸巾状态机 (TissuePaperState)

| 状态 | 进入条件 | 退出条件 |
|------|----------|----------|
| Connected | 初始状态 | 切断操作 |
| Cut | 切断操作 | 收回完成 |
| Retract | 切断后 | 收回中 |
| Retracted | 收回完成 | 重置 |

### 4.4 堆积纸巾状态机 (TissuePileState)

| 状态 | 进入条件 | 退出条件 |
|------|----------|----------|
| Idle | 初始状态/清除完成 | 添加纸巾 |
| Piling | 添加纸巾 | 0.1 秒后 |
| Clearing | 清除操作 | 动画完成 |

## 5. 判定区域设计

### 5.1 区域划分

```
┌─────────────────────────────┐
│                             │
│     ┌─────────────┐         │
│     │  TissueBox  │         │  ← 判定区域 1
│     │   Zone      │         │
│     └─────────────┘         │
│                             │
│         (纸巾延伸)           │
│                             │
│     ┌─────────────┐         │
│     │TissuePile   │         │  ← 判定区域 2
│     │   Zone      │         │
│     └─────────────┘         │
│                             │
└─────────────────────────────┘
```

### 5.2 判定区域配置

| 区域 | 组件类型 | 推荐尺寸 | 位置 |
|------|----------|----------|------|
| TissueBoxZone | BoxCollider2D | X=2.5, Y=2.5 | 纸巾筒位置 |
| TissuePileZone | BoxCollider2D | X=4, Y=5 | 场景下方 |

### 5.3 判定逻辑

```csharp
// 检测点击位置
if (tissueBoxZone.OverlapPoint(worldPos))
{
    // 纸巾筒区域操作
}
else if (tissuePileZone.OverlapPoint(worldPos))
{
    // 堆积区域操作
}
```

## 6. 数据配置

### 6.1 TissueGameConfig 参数

#### 数量设置
| 参数 | 默认值 | 说明 |
|------|--------|------|
| maxTissueCount | 15 | 纸巾堆积上限 |
| maxPaperLength | 20 | 纸巾最大延伸长度 |

#### 输入设置
| 参数 | 默认值 | 说明 |
|------|--------|------|
| pullThreshold | 50 | 向下划动触发阈值（像素） |
| swipeThreshold | 50 | 横向划动触发阈值（像素） |
| swipesPerClear | 3 | 清理一个纸巾需要的划动次数 |

#### 动画设置
| 参数 | 默认值 | 说明 |
|------|--------|------|
| clearMoveDistance | 1000 | 清除时向右移动距离 |
| clearMoveDuration | 0.5 | 清除动画时长（秒） |
| clearChamberDuration | 0.3 | 清空弹仓动画时长（秒） |
| reloadDuration | 0.5 | 装填新纸卷动画时长（秒） |

#### 延伸设置
| 参数 | 默认值 | 说明 |
|------|--------|------|
| segmentsPerPull | 1 | 每次向下划动延伸的纸巾段数 |
| segmentHeight | 0.1 | 每段纸巾的高度 |

#### 得分设置
| 参数 | 默认值 | 说明 |
|------|--------|------|
| idealTissueLength | 10 | 理想纸巾长度（段数） |

### 6.2 得分计算

```
得分 = f(切断长度，理想长度)

| 条件 | 得分 |
|------|------|
| 长度 = 理想长度 | 100 分（完美） |
| |长度 - 理想长度| ≤ 2 | 50 分（很好） |
| |长度 - 理想长度| ≤ 5 | 25 分（一般） |
| |长度 - 理想长度| > 5 | 10 分（较差） |
```

## 8. 游戏结束条件

### 8.1 失败条件
- 堆积纸巾数量达到上限（maxTissueCount = 15）

### 8.2 游戏结束处理
1. 触发 OnGameEnd 事件
2. 纸巾筒上移并消失
3. 显示最终得分

## 9. 技术实现

### 9.1 关键技术点

1. **多判定区域输入处理**
   - 使用 Collider2D.OverlapPoint() 检测点击位置
   - 支持鼠标和触摸统一处理

2. **状态机管理**
   - 每个组件独立状态机
   - 主管理器协调全局状态

3. **事件驱动架构**
   - 组件间通过事件通信
   - 松耦合设计

4. **ScriptableObject 配置**
   - 游戏参数可配置化
   - 便于平衡调整

### 9.2 性能优化

1. **对象池**（可扩展）
   - 纸巾段预制体复用
   - 减少 Instantiate/Destroy 开销

2. **协程动画**
   - 使用 StartCoroutine 处理时序动画
   - 避免 Update 中的复杂计算

3. **事件订阅管理**
   - OnEnable 订阅，OnDisable 取消订阅
   - 防止内存泄漏
